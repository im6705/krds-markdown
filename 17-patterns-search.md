# 검색 서비스 패턴

검색은 사용자가 큰 데이터 집합에서 원하는 정보를 찾을 수 있도록 도와주는 기능이다. 사용자가 무엇을 찾고 싶은지 알고 있는 경우에는 일차적인 정보 탐색 수단으로 사용될 수 있으며, 탐색 수단을 통해 원하는 콘텐츠를 찾지 못하는 상황에는 특정 정보와 관련된 단서를 제공함으로써 사용자가 필요한 콘텐츠를 쉽게 찾을 수 있도록 해준다.

**사용하기 적합하지 않은 경우**: 단일 화면 웹사이트나 서비스 또는 규모가 매우 작은 서비스에서는 통합 검색 없이 정보를 탐색할 수 있다.

---

## 1. 검색 유형

| 유형 | 설명 | 배치 |
|---|---|---|
| **통합검색** | 웹사이트 또는 애플리케이션 전체를 검색. 서비스에 익숙하지 않은 사용자에게 적합 | 헤더 영역, 결과는 별도 화면 |
| **부분 검색** | 소규모 데이터 집합(특정 목록, 표 등)을 검색 | 데이터 집합 상단, 결과는 데이터 집합에 직접 반영 |

---

## 2. 이용 상황별 플로(Flow)

| 단계 | 과정 | 세부 |
|---|---|---|
| 1 | **검색어 입력** | 검색어 입력 도움 받기, 검색어 입력과 수정 |
| 2 | **검색 결과 확인** | 결과 화면 진입, 검색 결과 개괄, 목록 탐색, 목표 결과 찾기 |
| 3 | **검색 결과 이용** | 외부/연계 서비스 이동, 상세 내용 확인, 검색 결과 다운로드/저장 |
| 4 | **재검색** | 검색 조건 계속 이용, 새로운 검색 조건 설정 |
| 5 | **상세 검색** | 상세 검색 기능 찾기, 상세 검색 조건 설정, 조건의 확인 및 적용 |

---

## 3. 단계별 상세

### 3.1 검색 기능 찾기

#### 구조

| # | 요소 | 설명 |
|---|---|---|
| 1 | **돋보기 아이콘** | 검색을 실행하거나 검색 레이어/화면을 실행하는 버튼 |
| 2 | **레이블** | 돋보기 아이콘에 대한 텍스트 레이블로 함께 버튼으로 작동 |

#### 사용성 가이드라인

| # | 가이드라인 | 적용 수준 |
|---|---|---|
| 01 | 서비스 내 모든 화면에서 통합 검색 기능을 실행할 수 있도록 한다 | 필수 |
| 02 | 통합 검색 기능 실행 버튼은 모든 화면에서 헤더 우측에 일관성 있게 배치한다 | 권장 |
| 03 | 통합 검색 기능은 검색어 입력 필드와 돋보기 버튼의 조합 또는 돋보기 버튼 형태로 사용한다 | 권장 |
| 04 | 부분 검색 기능은 콘텐츠 목록의 수준과 맥락을 고려하여 사용자가 예측할 수 있는 영역에 배치한다 | 권장 |
| 05 | 검색 기능 주변에 불필요한 요소의 배치를 최소화한다 | 권장 |

#### 접근성 가이드라인

- 돋보기 아이콘은 키보드 및 스크린 리더로 접근 및 조작할 수 있도록 한다 (KWCAG 2.2 키보드 사용 보장, WCAG 2.1 Keyboard (A))
- 돋보기 아이콘에 적절한 이름을 제공한다 — '돋보기'가 아닌 **'검색'** 또는 **'통합 검색'** (KWCAG 2.2 적절한 링크 텍스트)

### 3.2 검색어 입력

#### 구조

| # | 요소 | 설명 |
|---|---|---|
| 1 | **검색어 입력 필드** | 사용자가 검색어를 입력하는 텍스트 입력 영역 |
| 2 | **레이블** | 검색할 수 있는 내용을 암시하는 짧은 텍스트 |
| 3 | **돋보기 아이콘** | 검색을 실행하는 버튼 |
| 4 | **플레이스홀더** (선택) | 검색을 유도하기 위한 문구 |
| 5 | **검색어 삭제 버튼** | 입력한 검색어 전체를 삭제하는 기능 |
| 6 | **스코프 필터** (선택) | 특정 정보 범주/섹션으로 결과를 제한 |
| 7 | **실시간 검색어 제안** (선택) | 최근/인기 검색어, 입력 기반 검색어 제안 |
| 8 | **검색 도움말** (선택) | 검색어 입력 방식 예제나 도움말 |

#### 사용성 가이드라인

| # | 가이드라인 | 적용 수준 |
|---|---|---|
| 01 | 사용자가 검색어 입력 행위에 집중할 수 있도록 한다 (나머지 영역 명도/불투명도 낮춤) | 우수 |
| 02 | 스코프 필터를 사용하는 경우, '전체' 옵션을 제공한다 | 필수 |
| 03 | 검색어 입력 필드는 일반적인 검색어가 한눈에 파악될 수 있는 너비로 제공하고 줄바꿈이나 말줄임표를 사용하지 않는다 | 권장 |
| 04 | 통합 검색이 아닌 검색에는 입력 필드에 레이블을 제공한다 | 필수 |
| 05 | 플레이스홀더를 사용하는 경우, 유도 문구 외에 검색에 대한 단서나 레이블을 포함해서는 안 된다 | 필수 |
| 06 | 사용자에게 다양한 방식의 검색어 입력 도움을 제공한다 (검색어 예제, 인기 검색어, 이전 검색어, 첫 단어 제안 등) | 권장 |
| 07 | 검색어 입력 도움은 가능한 사용자가 검색어를 입력하는 시점에 실시간으로 제공한다 | 우수 |
| 08 | 실시간으로 검색어를 제안할 수 없는 경우, 검색 결과 화면에서 보조적인 도움 수단을 제공한다 | 권장 |

#### 접근성 가이드라인

- 검색어 입력/실행 관련 컨트롤 요소에 적절한 이름을 제공한다 (KWCAG 2.2 레이블 제공, WCAG 2.1 Name, Role, Value (A))
- 검색어 입력/실행 관련 대화형 요소는 키보드로 접근과 조작이 가능하도록 한다 (WCAG 2.1 Keyboard (A))
- 자동완성이나 추천 검색 정보에 보조 기술이 접근할 수 있도록 표현한다
- 실시간 검색어 입력 도움창의 출현 여부 및 추천 검색어를 스크린 리더가 탐지할 수 있도록 한다 (WCAG 2.1 Status Messages (AA))
- 실시간 검색어 입력 도움창은 검색어 입력 필드의 다음 요소로 제공한다 (KWCAG 2.2 콘텐츠의 선형화)
- 자동완성이나 추천 검색어 목록을 계층화하여 제공한다 (WCAG 2.1 Info and Relationships)

#### 상호작용 가이드라인

**검색어 입력 필드:**

| 동작 | 설명 |
|---|---|
| **Focusin** | 아무 값이 없으면 인기 검색어와 이전 검색 기록 제공 |
| **Keyup** | 한 글자 이상 입력 시 검색어 제안으로 변경, 검색어 삭제 버튼 제공 |
| **Enter** | 검색 실행 |
| **방향키 ↓** | 검색어 제안 목록의 첫 번째 요소로 초점 이동 |

**실시간 검색어 입력 도움:**

| 동작 | 설명 |
|---|---|
| **방향키 ↑, ↓** | 제안 목록 순회 (순환식) |
| **Click** | 선택된 항목으로 검색 실행 |
| **Enter** | 초점 가진 항목으로 검색 실행 |
| **Esc** | 도움창 닫힘, 초점이 입력 필드로 이동 |

#### 플랫폼 고려 사항

- 음성입력 버튼을 제공한다 (키보드 입력이 어려운 상황, 디지털취약계층 대응)
- 상세 검색 컨트롤이 많아 복잡할 경우 필수조건을 제외한 컨트롤은 모달/바텀시트로 숨김 처리 가능

### 3.3 검색 결과 확인

#### 구조

| # | 요소 | 설명 |
|---|---|---|
| 1 | **주제 탐색 탭** | 검색 결과 목록을 검색 주제별로 탐색할 수 있는 수단 |
| 2 | **결과 수** | 전체 검색 결과의 양을 나타내는 지표 |
| 3 | **결과 목록** | 검색 결과로 반환된 데이터 목록 |

#### 사용성 가이드라인

| # | 가이드라인 | 적용 수준 |
|---|---|---|
| 01 | 검색 결과가 표시되기 전에 로딩 상태에 대한 정보를 제공한다 (스피너/스켈레톤) | 우수 |
| 02 | 검색 결과 화면에 검색어 입력 필드를 유지한다 | 필수 |
| 03 | 검색 결과 화면에 여러 개의 검색어 입력 필드를 사용하지 않는다 | 권장 |
| 04 | 사용자가 재검색을 시도하지 않는 한 검색어 입력 필드에 검색어를 유지한다 | 권장 |
| 05 | 검색 결과 수를 표시한다 (주제별 탭 사용 시 주제별 결과 수도 포함) | 필수 |
| 06 | 검색어와 관련된 결과가 반환되는지 확인한다 (정기 점검) | 필수 |
| 07 | 유사 검색어나 관련된 검색어에 대한 검색 결과를 함께 제공한다 | 우수 |
| 08 | 초기 검색 결과 목록은 정확도 또는 관련도순으로 정렬하여 제공한다 | 권장 |
| 09 | 검색 결과가 지나치게 많이 제공되는 경우 범위를 제한할 수 있는 방법에 대해 안내한다 | 우수 |
| 10 | 검색 결과가 없는 경우, 결과를 명확하게 알리고 적절한 대안을 제공한다 | 필수 |

**검색 결과 없음 시 필수 표시 항목:**

- 결과 수: "0"
- 안내 메시지: "검색어와 일치하는 결과를 찾을 수 없습니다."
- 대안: 검색어 철자 확인, 다른 검색어 사용, 더 일반적인 검색어 사용, 검색 도움말 참조

### 3.4 상세 검색

#### 구조

| # | 요소 | 설명 |
|---|---|---|
| 1 | **정렬 컨트롤** | 검색 결과 목록의 순서를 변경하는 컨트롤 |
| 2 | **필터 컨트롤** | 특정 주제, 범주, 속성으로 결과를 제한하는 컨트롤 |
| 3 | **고급 검색 버튼/링크** | 복잡한 검색 조건 설정에 접근할 수 있는 수단 |

#### 검색 유형 선택 기준

| 구분 | 기본 검색 | 복합 검색 | 고급 검색 |
|---|---|---|---|
| **검색 범위** | 전체 / 소규모 데이터 | 전체 | 전체 / 대량 아카이브 |
| **메타 데이터 복잡성** | 유형 수와 주제 수 적음 | — | — |
| **검색 결과 정확도** | 1~2페이지 내 발견 가능 | — | — |
| **필터 유형** | — | — | 쿼리 연산자가 필요한 텍스트 필터 |

#### 사용성 가이드라인

| # | 가이드라인 | 적용 수준 |
|---|---|---|
| 01 | 기본 정렬, 필터 컨트롤을 제공한다 (통합검색: 검색 기간 필터 + 정렬 컨트롤) | 필수 |
| 02 | 기본 검색에서 정렬, 필터 컨트롤은 일정한 순서로 제공되어야 한다 | 권장 |
| 03 | 스코프 필터를 사용하는 경우 결과 페이지 상단에 설정된 범위를 명시적으로 표시한다 | 필수 |
| 04 | 정렬, 필터 적용 결과는 가능한 한 사용자가 값을 선택하였을 때 적용한다 | 우수 |
| 05 | 조건이 적용되었음을 확인할 수 있는 명확한 시각적 단서를 제공한다 | 우수 |

**통합검색 기본 컨트롤 옵션:**

- 검색 기간 필터: 사용자 정의 + 3개 이상 사전 정의 기간 (예: 오늘/일주일/1개월)
- 기본 정렬: 정확도/관련도순, 최신순, 인기순

#### 접근성 가이드라인

- 조건이 적용되었음을 확인할 수 있는 명확한 프로그램적 단서를 제공한다 — `aria-live="polite"` 사용 (WCAG 2.1 Status Messages (AA))

### 3.5 검색 결과 이용

#### 구조

| # | 요소 | 설명 |
|---|---|---|
| 1 | **주제 탐색 탭** | 검색 주제별 탐색 수단 |
| 2 | **배지** (선택) | 범주 관련 메타 데이터 표시 |
| 3 | **날짜** | 등록 일자 또는 최종 업데이트 일자 |
| 4 | **제목** | 본문 제목, 첨부 파일명, 메뉴명 등 |
| 5 | **설명** | 결과 정보에 대한 설명 |
| 6 | **인덱스** | 정보 구조 위계 표시 |
| 7 | **URL** (선택) | 외부 사이트로 연결되는 경우 링크 주소 표시 |
| 8 | **액션 버튼/링크** (선택) | 결과 화면 이동 또는 관련 기능 실행 |

#### 사용성 가이드라인

| # | 가이드라인 | 적용 수준 |
|---|---|---|
| 01 | 사용자가 필요로 하는 주제별로 검색 결과가 제공되도록 설계한다 | 필수 |
| 02 | 검색 주제 분류는 7개를 초과하지 않도록 한다 | 권장 |
| 03 | 주제별 탐색 탭을 사용하는 경우 전체 목록이 가장 첫 번째 탭으로 제공되어야 한다 | 권장 |
| 04 | 전체 결과 목록은 주제별 섹션 구분 없이 결과의 정확도나 관련도 순으로 정렬한다 | 우수 |
| 05 | 제목과 설명은 명확하고 간결하게 제공한다 (2줄 이내) | 권장 |
| 06 | 항목에 검색 결과 범주 정보가 명확하게 드러나도록 표현한다 | 권장 |
| 07 | 검색어와 일치하는 항목을 강조 표시한다 | 필수 |
| 08 | 기본 검색 결과 목록에 표시되는 항목의 수를 10개 이상으로 제공한다 | 권장 |

**통합 검색에서 제공 가능한 검색 주제:** 메뉴, 자주 묻는 질문, 뉴스/공지/소식, 첨부파일 등 자료, 정책/고시/법령, 이미지/동영상 미디어, 직원

### 3.6 재검색

#### 구조

| # | 요소 | 설명 |
|---|---|---|
