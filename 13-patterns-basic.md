# 기본 패턴

기본 패턴은 컴포넌트 요소들이 조합되어 핵심 과업을 수행하는 데 반복적으로 함께 사용되는 사용자 인터페이스 집합에 대한 가이드이다.

---

## 1. 개인 식별 정보 입력

개인 식별 정보는 사용자 본인을 포함하여 특정한 개인의 신원을 밝히거나 개인/단체에 대한 기본 정보 확인에 사용되는 모든 정보를 의미한다.

### 유형
- **이름**: 텍스트 입력 필드
- **생년월일**: YYYY / MM / DD 형식 입력
- **성별**: 남성 / 여성 / 선택 안 함 라디오 버튼
- **전화번호**: 010 / XXXX / XXXX 형식 입력

---

### 공통 사용성 가이드라인

| # | 가이드라인 |
|---|---|
| 01 | **개인 식별 정보의 수집이 필요한지 확인한다** - 서비스 이용을 위해 필요한 정보인지 재확인 |
| 02 | **입력이 필요한 데이터의 용도를 명확하게 설명한다** - 데이터 활용 방법과 편의성 안내 |
| 03 | **개인 식별 정보 입력 필드에 복사와 붙여넣기를 허용한다** - 입력에 어려움을 느끼는 사용자 고려 |
| 04 | **기본값을 설정하지 않는다** - 플레이스홀더 사용 시 실제 입력과 쉽게 구분 |

---

### 이름 입력 가이드라인

| # | 가이드라인 |
|---|---|
| 01 | **입력할 수 있는 모든 문자의 입력과 표시를 지원한다** - 기호, 숫자, 공백 포함. 본인인증 시 제약 사항을 명확히 설명 |
| 02 | **이름 입력 필드에 입력 가능한 글자 수를 적절하게 구성한다** - 한글 이름 기준 유효성 검사에서 최소 3글자 이상 입력을 요청하지 않도록 유의 |
| 03 | **이름 입력에 단일 텍스트 입력 필드를 사용한다** - 모든 사람의 이름이 성/이름 형식에 부합하지 않을 수 있음 |
| 04 | **개명한 사용자가 문제 없이 이름을 입력할 수 있는지 확인한다** - 개명 사용자를 위한 해결 방안 제공 |

#### 피해야 할 사례
- 이름 입력 필드가 5글자까지만 허용
- 최소 3글자부터 입력 가능한 유효성 검사 (2글자 이름 불가)

---

### 생년월일 입력 가이드라인

| # | 가이드라인 |
|---|---|
| 01 | **생년월일의 입력에 날짜 선택기(Calendar Picker)를 사용하지 않는다** - 이미 알고 있는 날짜를 선택하는 데 적합하지 않음 |
| 02 | **연도, 일의 입력에 셀렉트를 사용하지 않는다** - 직접 입력이 더 효율적 |
| 03 | **날짜 입력 형식을 정확하게 안내한다** - "생년월일 8자리를 입력해 주세요" |

#### 피해야 할 사례
- 생년월일 입력에 달력 팝업 제공

---

### 성별 입력 가이드라인

| # | 가이드라인 |
|---|---|
| 01 | **특정 성별을 기본값으로 지정하지 않는다** |
| 02 | **'선택 안 함' 옵션을 제공한다** |

---

### 전화번호 입력 가이드라인

| # | 가이드라인 |
|---|---|
| 01 | **활용 용도에 적합한 유형의 전화번호 입력을 요청한다** |
| 02 | **전화번호 입력 필드에 명확한 레이블을 제공한다** |
| 03 | **여러 가지 입력 형식에 유연한 입력 필드를 제공한다** |
| 04 | **국가 코드 입력에 콤보박스를 사용한다** |
| 05 | **여러 개의 전화번호 입력 요청 시, 각 번호를 구분할 수 있는 레이블을 제공한다** |
| 06 | **여러 개의 전화번호 입력 요청 시, 주로 사용하는 번호를 지정할 수 있는 옵션을 제공한다** |

---

## 2. 기본 패턴 전체 목록

PDF의 기본 패턴 장(Chap. V)에서는 다음 기본 패턴들이 정의되어 있다:

- **개인 식별 정보 입력** (이름, 생년월일, 성별, 전화번호)
- **도움** (안내 영역, 플레이스홀더, 인라인 텍스트, 툴팁, 맥락적 도움말, 도움 패널)
- **동의** (약관 동의, 개인정보 수집/이용 동의)
- **목록 탐색** (필터링/정렬, 페이지네이션, 무한 스크롤)
- **입력폼** (텍스트 입력, 날짜 입력, 파일 업로드)
- **확인** (입력 내용 확인, 결과 확인)
- **오류** (입력 오류, 시스템 오류, 접근 오류)

> 각 기본 패턴에는 사용성 가이드라인, 접근성 가이드라인, 상호작용 가이드라인, 플랫폼에 대한 고려 사항이 포함된다. 상세 내용은 원본 PDF의 해당 페이지를 참조한다.
