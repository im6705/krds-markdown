# 색상 시스템 (Color)

색상은 정보 전달과 시각적 경험을 결정하는 중요한 요소다. 사용자의 주의를 유도하고, 기능이나 메시지를 명확히 하여 직관적인 사용자 경험을 제공한다.

---

## 1. 색상 팔레트

색상 팔레트는 UI 전반에서 시각적 일관성을 유지하며 주 색상, 보조 색상, 상태 색상 등 다양한 역할을 효과적으로 구분할 수 있도록 설계한다.

### 팔레트 구성

- **HSL 컬러모델** 기준: 색상(Hue), 채도(Saturation), 명도(Lightness)
- **Gray**: 0(White) ~ 100(Black) 포함 **총 13단계**
- **다른 색상**: 0과 100 제외한 **11단계** (5% ~ 95%)

---

## 2. 접근성과 매직넘버

매직넘버는 색상 단계 간의 차이를 수치화하여 **WCAG 기준**에 따른 명도 대비를 쉽게 계산할 수 있게 도와준다.

### 매직넘버 정의

| 매직넘버 | 명도 대비 비율 | WCAG 기준 |
|---|---|---|
| **매직넘버 40** | 3:1 | 대형 텍스트/UI 컴포넌트 최소 대비 (WCAG AA) |
| **매직넘버 50** | 4.5:1 | 일반 텍스트 최소 대비 (WCAG AA) |
| **매직넘버 70** | 7:1 | 일반 텍스트 향상된 대비 (WCAG AAA) |
| **매직넘버 90** | 15:1 | 최고 수준 대비 |

### 매직넘버를 준수한 컬러 팔레트 만드는 방법

1. **HSL 컬러모델 설정** - 원하는 색상의 L값을 임의로 50%로 설정
2. **Lightness 11단계 제작** - L 50%를 기준으로 L값의 범주 11개 (5%~95%)
3. **매직넘버 50 준수 값으로 조정** - primary-50(L 50%)의 대비율 3.9:1이면 4.5:1 충족 불가이므로 L 55%로 변경
4. **매직넘버 40, 70 적용** - 3:1, 7:1 대비율 기준으로 나머지 컬러 조정

### 접근성 준수 주의 사항

- 명도 대비 기준을 고려하여 단계별 색상 설정, 매직 넘버로 대비 비율 정확히 설정
- **색맹 시뮬레이터 활용**: 색각 이상자를 위한 최적의 색상 보장
- 기본 모드/선명한 화면 모드에서 background 컬러와 비슷한 **색상 5, 95단계** 주의 사용

---

## 3. 모드 (기본 / 선명한 화면)

| 모드 | 설명 |
|---|---|
| **기본 모드** | 일반 사용자를 위한 표준 색상 모드 |
| **선명한 화면 모드** | 색상 대비를 높여 시각적 접근성을 강화한 모드 |

### 색약자를 위한 고려

- **적록색맹, 적록색약** (적색, 녹색) = 99%의 색각이상자
- **청색맹, 청색약** (청색, 녹색, 황색)
- 색상 외에 **텍스트와 아이콘**으로 정보를 보완
- 특정 색상 조합(적색+녹색) 피하거나 명도 차이 조절

---

## 4. 색상 시스템 구성

| 색상 범주 | 역할 | 용도 |
|---|---|---|
| **Gray** | 중립적 정보 | 배경, 구분선, 글자 |
| **Primary** | 주요 상호작용 요소 | 중요한 버튼, 링크 등 |
| **Secondary** | 보조 상호작용 요소 | 사이드 메뉴, 대부분의 UI 요소 등 |
| **Accent** | 강조 요소 (제한적 사용) | 새로운 알림, 중요 알림 배지 등 |
| **Graphic** | 보조 요소 | 그래프, 배너, 일러스트 등 |
| **System** | 상태 전달 | Danger, Warning, Success, Info |

### 주요 색상 (Primary, Secondary, Gray)

- **Primary**: 정부 청색 기반. 정부의 상징성, 주요 액션/중요 정보 강조. 명도/채도를 높여 시각적으로 눈에 띄도록
- **Secondary**: 정부 회색/적색 기반. 보조적 역할, 덜 중요한 액션, 배경 구분. 신뢰감/안정감 있는 톤
- **Gray**: 블루 그레이 계열. 배경, 텍스트, 구분선. 중립적이며 콘텐츠에 초점

#### Secondary 사용 시 주의 사항
- secondary-70은 선명한 화면 모드에서 다른 색상과 시각적 구분이 어려움
- 어두운 배경 모드에서 색각이상자에게도 구분되는 유사 색상의 밝은 톤으로 적용

---

## 5. 색상 비율 (60-30-10 원칙)

| 비율 | 역할 | 용도 |
|---|---|---|
| **60%** | 배경 및 중립적인 색상 | surface, background, text, border, divider |
| **30%** | 보조 색상 | element, icon, divider, action icon |
| **10%** | 주요 기능/중요 상호작용 색상 | element, icon link, button, input, text |
| **최대 5%** | 강조 색상 (accent color) | 중요한 강조 요소에만 제한적 사용 |

> **주의**: 강조 색상은 최대 5%를 넘기지 않는다. 너무 많은 색상은 색맹 사용자가 인식하기 어려움.

---

## 6. 투명도 (Alpha)

- **5단계** 투명도 사용
- BG, Text, Line에 사용 가능
- 텍스트는 가독성을 위해 **배경과의 명암비를 반드시 고려**

---

## 7. 시스템 색상

시스템 색상은 상태를 직관적으로 전달하며, **아이콘이나 텍스트를 함께 사용**하여 시각적 혼동을 최소화한다.

### 시스템 색상 토큰 매핑

| 상태 | 색상 계열 | 기본 모드 | 선명한 화면 모드 |
|---|---|---|---|
| **Danger (위험)** | 빨간색 계열 | 아이콘: danger-50, 텍스트: danger-60, 배경: danger-5, 보더: danger-10 | 아이콘: danger-20, 텍스트: danger-20, 배경: danger-95, 보더: danger-90 |
| **Warning (주의)** | 노란색 계열 | 아이콘: warning-50, 텍스트: warning-60, 배경: warning-5, 보더: warning-10 | 아이콘: warning-20, 텍스트: warning-20, 배경: warning-95, 보더: warning-90 |
| **Success (성공)** | 녹색 계열 | 아이콘: success-50, 텍스트: success-60, 배경: success-5, 보더: success-10 | 아이콘: success-20, 텍스트: success-20, 배경: success-95, 보더: success-90 |
| **Information (안내)** | 파란색 계열 | 아이콘: information-50, 텍스트: information-60, 배경: information-5, 보더: information-10 | 아이콘: information-20, 텍스트: information-20, 배경: information-95, 보더: information-90 |

---

## 8. 색상 적용 규칙

### 배경/표면 색상 (Background, Surface)

Elevation에 따라 단계적으로 색상을 조절한다.

| 모드 | elevation-1 | elevation 0 | elevation+1 | elevation+2 |
|---|---|---|---|---|
| **기본 모드** | gray-10 | gray-0 | gray-5 | gray-0 |
| **선명한 화면 모드** | gray-90 | gray-100 | gray-95 | gray-90 |

- 기본 모드: 흰색 surface와 색상 surface를 번갈아 사용, border/shadow로 고도 표현
- 선명한 화면 모드: 레이어가 쌓일수록 점차 밝아지게 표현

### 글자 색상 매직넘버

| 모드 | 최소 대비 | 본문 대비 |
|---|---|---|
| **기본 모드** | 매직넘버 40 (3:1) | 매직넘버 50 (4.5:1) |
| **선명한 화면 모드** | 매직넘버 70 (7:1) | 매직넘버 90 (15:1) |

### 인풋 보더 색상

| 모드 | 매직넘버 |
|---|---|
| **기본 모드** | 매직넘버 50 |
| **선명한 화면 모드** | 매직넘버 50 (두께 변화) |

### 아이콘 색상

| 모드 | 매직넘버 |
|---|---|
| **기본 모드** | 매직넘버 40 |
| **선명한 화면 모드** | 매직넘버 70 또는 매직넘버 50 (두께 변화) |

### 상호작용 상태 값

- 상호작용이 있는 요소의 상태 값은 **점진적인 증가 또는 감소**의 규칙 사용
- 배경이 있는 경우: 기준 색상 앞뒤 레벨 적용

---

## 9. 전체 색상 팔레트 관리 규칙

- 주요 색상 외에 색상이 필요할 경우 **색상 팔레트에 있는 색상 값을 우선으로 사용** 권장
- 정의되지 않은 새로운 색상은 **재사용성이 높을 경우에만** 팔레트에 추가
- 확장형 스타일 기관: 독자적 고유 컬러 정의 가능하지만, **표준 팔레트 색상 단계 및 규칙은 필수로 참고**

---

## 10. 사용 가이드 (모범 사례 / 피해야 할 사례)

### 시스템 색상
- **모범 사례**: 시스템 색상은 시각적 보조수단(아이콘, 텍스트)과 함께 사용
- **피해야 할 사례**: 색상만으로 정보 전달

### 주요 색상 사용
- **모범 사례**: 주요 액션이나 중요한 정보를 강조하는 데 사용
- **피해야 할 사례**: 주요 색상을 과도하게 사용하여 강조 효과 감소

### 접근성 고려한 대비율
- **모범 사례**: 명도 대비를 준수
- **피해야 할 사례**: 명도 대비를 무시한 색상 조합

### 상태 값 색상
- **모범 사례**: 동일한 색조(hue) 내에서 상태 변화 표현
- **피해야 할 사례**: 상태 값에 다른 색조 사용

### 투명도 (Alpha)
- **모범 사례**: 적절한 투명도로 시각적 계층 표현
- **피해야 할 사례**: disabled 상태에 투명도 색상 사용하지 않는다
