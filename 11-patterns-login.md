# 로그인 서비스 패턴

로그인은 사용자의 신원을 확인하는 과정으로 사용자가 서비스에 접근할 수 있도록 하는 수단이다. 사용자에게 개인화된 경험을 제공하거나 사용자의 신분/신원을 인증하고자 하는 경우에 사용하기 적합하다.

---

## 1. 인증 유형

| 유형 | 설명 | 예시 |
|---|---|---|
| **지식 기반 인증** | 사용자만이 유일하게 알고 있는 내용에 기반한 사용자 확인 방법 | 아이디/패스워드, PIN |
| **소유 기반 인증** | 사용자가 보유하고 있는 품목을 활용한 인증 방법 | 토큰, 공동인증, 간편인증, 문자 인증 |
| **생체 기반 인증** | 사용자의 생체 정보, 행동에 기반한 인증 방법 | 지문 인증, 페이스 아이디 |
| **다중 요소 인증** | 여러 가지 인증 방식을 결합하여 제공하는 인증 방법 | 복합 인증 |

---

## 2. 이용 상황별 플로(Flow)

| 단계 | 과정 | 세부 |
|---|---|---|
| 1 | **로그인 방식 확인 및 선택** | 로그인 방식 탐색: 2개 이상의 로그인 방식에 대해 비교하고 선택. 도움말 확인: 로그인 방식/방법에 대한 안내 확인 |
| 2 | **로그인 정보 입력** | 인증 방식에 따라 정보를 입력하는 과정 |
| 3 | **로그인 완료** | 로그인이 성공적으로 완료되어 화면을 벗어나는 과정 |
| 4 | **로그아웃** | 인증을 해제하는 과정. 사용자가 자발적으로 수행하거나 세션 유지 시간 만료로 자동 해제 |

---

## 3. 단계별 상세

### 3.1 로그인 기능 찾기

#### 구조

| # | 요소 | 설명 |
|---|---|---|
| 1 | **아이콘** | 로그인에 사용되는 링크 요소임을 보다 직관적으로 인지할 수 있도록 제공되는 아이콘 |
| 2 | **레이블** | 아이콘에 대한 텍스트 레이블 |

#### 사용성 가이드라인

| # | 가이드라인 | 적용 수준 |
|---|---|---|
| 01 | 로그인 링크는 모든 화면에서 일관된 위치에 주목도 있게 배치한다. 로그인으로 이동할 수 있는 링크가 메인 화면 또는 일부 화면에만 제공될 경우, 로그인이 필요한 사용자는 특정 화면으로 이동하는 경우에만 기능을 이용할 수 있어 이용에 어려움을 겪게 된다 | 필수 |
| 02 | 헤더에서 제공되는 로그인 링크는 사용자 아이콘과 레이블로 구성한다. 아이콘만 단독으로 사용되는 경우 사용자에 따라 링크의 목적을 빠르게 이해하기 어려울 수 있으므로 텍스트 레이블을 함께 제공하는 것이 좋다 | 권장 |
| 03 | 로그인 링크는 항상 '로그인' 화면으로 연결되어야 한다. 로그인 링크의 목적지는 로그인 프로세스를 시작하는 화면 또는 로그인할 수 있는 다양한 계정이 있는 계정 선택 화면으로 설정해야 한다 | 필수 |

#### 접근성 가이드라인

- 로그인 링크는 스크린 리더에서 링크로 인지될 수 있도록 한다. 로그인 링크를 버튼 요소로 마크업하게 되면 로그인 화면으로 이동하는 동작이 발생함을 예측할 수 없다 (WCAG 2.1 Name, Role, Value (A))

#### 플랫폼에 대한 고려 사항

- 화면 너비가 충분하지 않은 경우에도 로그인 링크를 직관적으로 인지할 수 있는 형태로 표현한다
- 작은 화면 너비에서도 가능한 한 로그인 링크를 숨기지 않고 헤더나 탭바 영역에 로그인/마이페이지 링크를 상시 배치한다
- 만약 서비스 이용에 로그인이 중요하지 않다면 메뉴 레이어에서만 로그인 링크를 제공할 수 있다
- 서비스 중심의 모바일 앱에서는 탭바 하단에 상시 노출하여 표시한다
- 웹과 모바일을 일원화하여 사용할 경우 상단에 배치할 수 있다

### 3.2 로그인 안내

#### 구조

| # | 요소 | 설명 |
|---|---|---|
| 1 | **오버레이(Overlay)** | 로그인 안내 모달과 하단의 기본 창을 시각적으로 구분하기 위한 그림자 또는 가림막 |
| 2 | **헤더** | 모달의 제목을 포함한 영역으로 제목 텍스트를 통해 로그인 화면으로의 전환이 필요함을 안내 |
| 3 | **닫기 버튼** (선택) | 사용자가 모달을 닫을 수 있게 하는 버튼 요소 |
| 4 | **본문** | 정보와 다른 컴포넌트 요소가 제공되는 영역 |
| 5 | **푸터** | 모달의 하단 영역으로 로그인 안 함 옵션, 로그인 화면으로 이동하는 링크 등의 액션 버튼으로 구성 |

#### 유형

| 유형 | 설명 |
|---|---|
| **기본** | 로그인이 필수적인 화면/기능에 접근을 시도하는 경우에 사용 |
| **추천** | 로그인이 필수적인 화면/기능에 접근을 시도하는 상황에서 로그인 방식에 따라 이후 서비스 이용 과정이 달라지는 경우에 사용 |
| **제안** | 로그인이 필수적인 화면/기능은 아니지만 로그인을 했을 때의 이점이 존재하여 로그인할 것을 제안하는 경우에 사용 |

#### 사용성 가이드라인

| # | 가이드라인 | 적용 수준 |
|---|---|---|
| 01 | 로그인에 대한 안내는 모달로 제공하여 사용자의 과업 맥락이 유지될 수 있도록 한다. 로그인하지 않는 옵션을 선택하여 탐색 중인 화면에 머무르고자 하는 사용자가 현재의 이용 맥락이 유지됨을 인지할 수 있도록 모달 레이아웃을 이용하여 안내를 제공한다 | 권장 |
| 02 | 로그인은 꼭 필요한 상황에서 유도한다. 기본/추천 유형에서는 사용자가 로그인이 필수적인 화면/기능에 접근하기 위한 액션 버튼을 실행하였을 때 제공한다. 제안 유형에서는 사용자가 접근을 시도한 화면이 로딩된 후에 제공한다 | 권장 |
| 03 | 로그인을 제안하는 경우 모달 본문에서 로그인을 통해 얻을 수 있는 이점에 대해 명확한 설명을 제공해야 한다. 로그인했을 때의 이점, 로그인하지 않았을 때의 어려움을 포함하여 충분한 설명을 제공해야 한다 | 우수 |
| 04 | 사용자가 이용 절차를 단축할 수 있는 로그인 방식을 추천한다. 로그인 방식에 따라 서비스 이용 방식이 달라지는 경우 이용 절차를 단축할 수 있는 로그인 방식을 추천한다 | 우수 |
| 05 | 로그인 안내 모달에는 '로그인 안 함' 옵션을 제공하여 원하지 않는 이동 동작이 발생하지 않도록 한다. 현재 화면에 머무르기로 결정한 사용자가 서비스를 계속 이용할 수 있도록 모달 푸터에 '로그인 안 함' 옵션을 제공해야 한다 | 권장 |

### 3.3 로그인 방식 확인/선택

#### 구조

| # | 요소 | 설명 |
|---|---|---|
| 1 | **부제목** | 특정 기관/서비스의 로그인 화면임을 안내하는 제목 |
| 2 | **제목** | 이용 목적을 선택하도록 유도하는 제목 텍스트 |
| 3 | **설명** | 로그인 목적 선택에 대한 이해를 돕기 위해 제공되는 설명 텍스트 |
| 4 | **방식 선택 목록** | 사용자의 유형, 목적에 따라 로그인 화면을 분리하여 달성 과업에 필요한 로그인 작업에 집중할 수 있도록 도움 |
| 5 | **방식 선택 탭** | 통합 로그인과 기존 서비스 로그인을 분리하여 사용자 혼란을 방지 |
| 6 | **섹션 제목** | 인증 후 이용 가능한 서비스 범위를 식별하는 텍스트 |
| 7 | **방식 선택 목록** | 인증 후 이용 가능한 서비스 범위에 따라 로그인 방식을 선택할 수 있는 목록 |
| 8 | **디스클로저** | 로그인 방식에 대한 세부 정보, 참고 사항 등을 나타내거나 숨기기 위한 컨트롤 |
| 9 | **로그인 관련 도움말** | 로그인 관련 질문 및 문제 해결을 위한 도움말을 확인할 수 있는 화면으로 이동하는 링크 |
| 10 | **회원 가입 링크** | 계정 생성을 위한 화면으로 연결되는 링크 |

#### 유형

**레이아웃 유형**

| 구분 | 목록 | 탭 |
|---|---|---|
| 인증 유형 | - | 사용자만이 유일하게 알고 있는 내용에 기반한 사용자 확인 방법 (아이디/패스워드, PIN) |
| 인증 방식의 다양성 | 전체 인증 방식의 수가 15개 이하 | 전체 인증 방식의 수가 15개를 초과 |

**절차 유형**

| 구분 | 단일형 | 단계형 | 집중형 |
|---|---|---|---|
| 인증 유형 | 인증 주체의 구분 없음 | 인증의 주체나 로그인 방식에 구분이 있음 | 인증 유형에 상관없이 단계형 로그인에 사용 |
| 인증 권한 수준 | 모든 인증 방식의 권한이 동일 | 인증 방식별로 권한이 다르고 차이가 서비스 이용에 영향을 미침 | - |
| 사용자 이용 빈도 | 관련 없음 | 이용 빈도 낮음 | 이용 빈도 높음 |

#### 사용성 가이드라인

| # | 가이드라인 | 적용 수준 |
|---|---|---|
| 01 | 빈번하게 사용되는 로그인 방식에 우선적으로 접근할 수 있는 수단을 제공한다. 사용자 데이터를 기반으로 보편적으로 많이 사용하거나 서비스 관점에서 추천하는 방식을 먼저 노출한다 | 우수 |
| 02 | 동일한 수준의 로그인 방식을 탭으로 구분하여 선택하게 하지 않는다. 탭은 두 개 이상의 콘텐츠 섹션을 비교하는 데 적합하지 않다. 로그인 주체, 방식이 명확하게 구분되는 경우에만 탭 사용 | 권장 |
| 03 | 다양한 로그인 방식에 대한 분류 체계에 사용자의 관점과 언어를 반영한다. 기술적인 관점이 아닌 사용자의 이용 목적에 따라 분류하여 제공한다 | 권장 |
| 04 | 로그인 방식에 따라 서비스 이용 절차가 달라지는 경우, 사전에 안내를 제공한다 | 권장 |
| 05 | 로그인 방식과 입력 항목에 대한 직관적인 안내를 제공한다. 안내 정보를 해당 항목 주변에 제공한다 | 권장 |
| 06 | 도움말 정보, 매뉴얼 링크를 제공한다. 처음 방문자, 익숙하지 않은 사용자를 위해 스스로 문제를 해결할 수 있는 링크를 제공한다 | 권장 |
| 07 | 도움말 정보는 사용자의 다양한 이용 환경을 고려하여 작성한다. 특정 화면 크기, 특정 디바이스에서만 참고할 수 있는 정보는 일부 사용자의 문제 해결에 도움을 주지 못한다 | 필수 |

#### 접근성 가이드라인

- 인지 기능 테스트에 의존하지 않는 인증 방식을 제공한다. 인증 과정이 인지 기능 테스트(비밀번호 입력, 패턴 인식, 임의 문자열 기억, 계산 수행 등)에 의존하는 경우, 인지 기능 테스트에 의존하지 않는 인증 방법을 적어도 하나 이상 제공해야 한다. 다만, 사용자의 이름이나 이메일 주소, 전화번호는 인지 기능 테스트로 간주하지 않는다 (KWCAG 2.2 접근 가능한 인증)

#### 플랫폼에 대한 고려 사항

- 로그인 방식을 우선으로 화면에 배치한다. 작은 화면 특성상 사용자가 로그인 방식의 전체를 한눈에 확인할 수 있도록 화면에 배치한다

### 3.4 로그인 정보 입력

#### 구조

| # | 요소 | 설명 |
|---|---|---|
| 1 | **레이블** | 계정 정보 입력 필드의 이름 |
| 2 | **계정 정보 입력 필드** | 계정 정보를 입력하기 위한 텍스트 입력 필드 |
| 3 | **로그인 정보 저장 옵션** | 다음 번 로그인 시 활용할 수 있는 계정 정보를 저장하기 위한 체크박스 옵션. 아이디, 비밀번호, 전체 계정 정보를 저장할 수 있음 |
| 4 | **아이디/비밀번호 찾기 링크** | 계정 정보를 기억하지 못하는 사용자가 아이디 조회 또는 비밀번호 재설정을 할 수 있도록 연결하는 링크 |
| 5 | **도움말** | 계정 정보 입력 과정 또는 로그인 방식에 대해 참고할 수 있는 정보 또는 관련 서비스 링크 |

#### 사용성 가이드라인

| # | 가이드라인 | 적용 수준 |
|---|---|---|
| 01 | 아이디 형식에 대한 단서를 사전에 제공한다. 이메일 주소를 아이디로 사용하는 경우 이메일 형식으로 입력해야 한다는 것을 안내한다. 레이블 및 도움말을 플레이스홀더가 아닌 레이블로 제공한다 | 필수 |
| 02 | 비밀번호를 표시할 수 있는 옵션을 제공한다. 기본적으로 비밀번호를 숨기되, '비밀번호 표시' 버튼을 통해 사용자가 비밀번호를 확인할 수 있도록 돕는다 | 필수 |
| 03 | 복사/붙여넣기를 제한하지 않는다. 계정 정보의 기억에 어려움이 있거나 키보드/키패드를 빠르고 정확하게 입력하기 어려운 사용자를 위해 복사/붙여넣기 기능을 제한하지 않아야 한다 | 필수 |
| 04 | 로그인 정보를 저장할 수 있는 옵션을 제공한다. 사용자의 최근 로그인 방식을 저장하거나 선호하는 로그인 방식을 직접 설정할 수 있게 하여 자주 방문하는 사용자의 로그인 경로를 단축한다 | 우수 |
| 05 | 아이디/비밀번호 찾기 링크를 제공한다. 대부분의 로그인 오류는 사용자가 계정 정보를 잊어버리는 것에서 기인한다. 로그인 양식 주변에 아이디/비밀번호 찾기 링크를 제공해야 한다 | 필수 |
| 06 | 오류 메시지는 명확하고 간결하게 제공한다. 사용자가 무엇이 잘못되었는지 이해할 수 있도록 도와주고 오류를 해결할 수 있는 방법을 안내해야 한다 | 필수 |

#### 접근성 가이드라인

- 초점이 논리적인 순서로 이동할 수 있도록 제공한다. 비밀번호 입력 양식 이전에 로그인 버튼으로 초점이 이동하거나, 로그인 버튼 이후에 아이디 및 비밀번호 저장 옵션으로 초점이 이동해서는 안 된다 (KWCAG 2.2 초점 이동과 표시, KWCAG 2.2 콘텐츠의 선형 구조, WCAG 2.1 Focus Order (A), WCAG 2.1 Meaningful Sequence (A))
- 입력 필드에 자동 완성 속성을 설정한다. 정보 입력에 필요한 사용자의 인지적, 신체적 노력을 최소화할 수 있도록 HTML 표준인 autocomplete 속성을 사용하여 입력 필드의 목적에 맞는 자동완성 정보를 사용자 에이전트에 전달한다 (WCAG 2.1 Identify Input Purpose (AA))

#### 상호작용 가이드라인

- 제출하기 전 가능한 한 많은 사용자 데이터의 유효성을 검사한다. 실시간 유효성 검사는 입력 필드가 포커스를 잃을 때 발생하며 유효하지 않은 문자나 빈 필드와 같은 입력 오류를 확인해야 한다
- 사용자가 로그인 폼을 제출하였을 때 오류가 발생한 경우 해당 항목으로 초점을 이동시킨다. 서버 측 오류가 발생하면 화면을 다시 로드하고 비밀번호 필드를 지운 다음 사용자 이름 입력 필드로 사용자를 돌려보내야 한다
- 인라인 메시지 등의 방식을 활용하여 오류를 표시하고 사용자가 문제를 해결하는 방법에 대한 명확한 지침을 제공한다

#### 플랫폼에 대한 고려 사항

- 생체인식을 사용할 경우 자동로그인 기능을 추가한다. 모바일 앱 특성상 사용자의 생체인식을 사용할 경우 자동로그인 기능을 사용할 수 있도록 한다
- 로그인 화면 하단 또는 성공 직후 안내 모달에서 로그인 유지 옵션을 제시하되, 기본값은 기관 보안정책에 따른다

| 5 | **로그아웃까지 남은 시간** | 로그인 세션이 유지되기까지 남은 시간을 표시 |

#### 유형

| 유형 | 설명 |
|---|---|
| **로그인 만료 안내 모달** | 사용자가 서비스를 이용하는 중에 전역적으로 제공되는 로그인 만료 안내로 모달 양식으로 제공 |
| **로그인 세션 유지 시간 안내** | 사용자의 이용 속도가 느린 화면/단계에서 로그인 세션 만료 시 데이터 손실이 발생하는 등 돌이킬 수 없는 결과로 이어질 수 있는 경우에 사전적으로 세션 유지 시간 제한에 대해 안내하여 오류를 예방하기 위한 목적으로 사용 |

#### 사용성 가이드라인

| # | 가이드라인 | 적용 수준 |
|---|---|---|
| 01 | 사용자에게 로그인 세션이 만료되기 전에 유지 시간에 제한이 있음을 안내한다. 서비스 이용 도중 로그아웃되는 상황이 발생할 수 있음을 사용자가 분명히 인지할 수 있도록 안내를 제공한다 | 필수 |
| 02 | 로그인 세션 시간 만료에 대한 안내는 모달로 제공하여 사용자의 과업 맥락이 유지될 수 있도록 한다. 탐색 중인 화면에 머무르고자 하는 사용자가 로그인 상태 연장 옵션을 선택했을 때 현재 화면이 유지됨을 인지할 수 있도록 한다 | 권장 |
| 03 | 로그인 세션 시간 만료에 대한 안내는 세션이 만료되기 최소 20초 전에 제공한다. 사용자에 따라 안내 내용을 확인하고 연장하기 버튼을 누르는 데 오랜 시간이 걸릴 수 있다. 시스템 반응 속도가 느린 경우 더 긴 시간을 제공하는 것이 적절하다 | 필수 |
| 04 | 로그인 만료 안내 모달에 시간 연장하기 버튼과 로그아웃하기 버튼을 제공한다. 서비스를 계속 이용하고자 하는 사용자와 이용을 종료하고자 하는 사용자가 다음 행동을 선택할 수 있는 수단을 제공한다 | 필수 |

#### 접근성 가이드라인

- 로그인 만료 안내 모달이 활성화된 상태에서 아무 키나 눌렀을 때 연장 가능하도록 한다. 사용자마다 시간 연장 버튼을 누르는 데 필요한 노력의 정도가 다를 수 있으므로 가능한 한 단순한 방법으로 로그인 연장 기능을 실행할 수 있는 방안을 제공한다 (KWCAG 2.2 응답시간 조절, WCAG 2.1 Timing Adjustable (A))
- 시간 연장은 최소 10회 이상 수행할 수 있도록 제공한다. 사용자에 따라 서비스 이용에 필요한 시간이 다를 수 있음을 고려한다 (KWCAG 2.2 응답시간 조절, WCAG 2.1 Timing Adjustable (A))

| 05 | 로그아웃 안내 화면에 사용자의 행동을 유도할 수 있는 액션 버튼을 제공한다. '다시 로그인하기' 버튼을 제공하고 로그인이 완료되었을 때 사용자가 탐색 중이던 화면으로 연결되어야 한다 | 우수 |
