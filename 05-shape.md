# 형태 / 래디어스 (Shape)

형태는 브랜드의 시각적 아이덴티티를 표현하는 중요한 요소로, UI 요소에 적용되는 둥글기 값인 래디어스(radius)를 통해 버튼, 카드, 컨테이너, 이미지 등에 고유한 느낌과 분위기를 만들어낸다.

---

## 1. 형태가 주는 이미지

### 각진 형태 (Angular/Square Shapes)
- **전문성**: 정확하고 전문적인 느낌, 신뢰감 전달
- **효율성**: 깔끔하고 효율적인 인상, 복잡한 정보를 깔끔하게 전달

### 둥근 형태 (Rounded Shapes)
- **친근함**: 부드럽고 친근한 인상, 따뜻한 사용자 경험 유도
- **안정감**: 보호받는 느낌 제공, 긍정적인 감정 유도

---

## 2. 래디어스 5단계 (표준형 스타일)

표준형 스타일은 **2px~12px** 범위이며, 최댓값은 **12px**.

| Level | Radius | Container Size | 용도 | 적용 컴포넌트 |
|---|---|---|---|---|
| **Xsmall** | 2px | 8x8, 12x12, 16x16 | 초소형 크기 (인디케이터, 배지, 프로그레스 바) | Element |
| **Small** | 4px | 20x20, 24x24, 32x32 | 작은 컴포넌트 | Chips, Checkbox, Radio button, Switch, Tag |
| **Medium** | 6px | 40x40, 48x48, 56x56, 64x64 | 기본 값 (버튼, 인풋) | Button, Text input, Textarea, Select, Carousel-Number, Step indicator, Pagination |
| **Large** | 10px | 72x72, 80x80 | 큰 컴포넌트 | Card, Dialog |
| **Xlarge** | 12px (max) | 96x96, 120x120 | 화면 비율이 큰 컴포넌트 | Banner, Dialog, Bottom sheet, Card |

> **참고**: 온전한 원형 형태를 사용하는 경우 `radius-max` 토큰을 사용한다.

---

## 3. 래디어스 계산법 (표준형)

### 공식

```
컨테이너 높이 x 비율(1/8 = 0.125) = radius
```

### 계산 후 반올림 규칙

1. radius 결과를 반올림
2. **홀수인 경우, 숫자가 더 높은 짝수로 변경**

### 계산 예시

| 컨테이너 높이 | 계산 (높이 x 0.125) | 적용 Radius |
|---|---|---|
| 12px | 12 x 0.125 = 1.5 | **2px** (반올림 후 짝수) |
| 20px | 20 x 0.125 = 2.5 | **4px** (반올림 후 짝수) |
| 40px | 40 x 0.125 = 5 | **6px** (짝수로 올림) |
| 120px | 120 x 0.125 = 15 | **12px** (max 값 적용) |

---

## 4. 확장형 스타일

- 각 기관의 아이덴티티에 맞게 **커스터마이즈된 래디어스 값** 설정 가능
- 1px 이상의 래디어스 값 사용 시 **표준형 스타일의 계층 구조를 참고**하여 일관성 유지

### 확장형 래디어스 비율 적용 방법

1. 기준이 되는 컴포넌트에 기관에 맞는 래디어스 값을 테스트하여 **비율 값**을 찾는다
2. 그 비율로 사용되는 컴포넌트 높이 값 기준으로 래디어스 값을 적용한다

### 규칙
- 래디어스 값은 **짝수**로 사용 권장
- 홀수이거나 소수점이 나오면 함께 사용될 컴포넌트를 고려하여 **근사치에 맞게 올림하여 짝수 값** 사용

---

## 5. 표현 방법

| 단위 | 사용 상황 |
|---|---|
| **px** | 버튼, 입력 필드 등 크기에 맞는 일관된 둥글기 설정이 중요한 요소 |
| **%** | 프로필 사진처럼 완전한 원형이 필요한 경우 (50% = 원형) |

### % 사용 시 주의 사항
- **비율 변동**: 컴포넌트마다 크기 비율이 다르므로 래디어스 값 조정 어려움
- **텍스트 일관성 부족**: UI 요소마다 래디어스 값이 일정하지 않아 일관성 유지 어려움
- **작은 컴포넌트**: 50% 적용 시 지나치게 둥글어져 의도와 다른 결과

---

## 6. 래디어스 적용 원칙

- 함께 사용되는 **비슷한 크기의 구성 요소는 래디어스 값을 동일**하게 적용
- 화면 구성 요소 간의 래디어스 조화 유지

---

## 7. 사용 가이드 (모범 사례 / 피해야 할 사례)

### 비율
- **모범 사례**: 컨테이너 사이즈가 커지면 래디어스 값도 비율에 맞게 적용
- **피해야 할 사례**: 컨테이너 크기와 무관하게 동일한 래디어스 사용

### 동일한 래디어스 적용
- **모범 사례**: 비슷한 크기의 구성 요소는 동일한 래디어스를 적용
- **피해야 할 사례**: 비슷한 크기의 구성 요소에 다른 래디어스 적용

### 콘텐츠 영역 침범
- **모범 사례**: 둥근 형태 사용 시 내부 패딩값을 충분히 제공
- **피해야 할 사례**: 과한 래디어스로 콘텐츠에 영향

### % 사용
- **모범 사례**: 원형이 필요한 경우에만 % 사용
- **피해야 할 사례**: 작은 컴포넌트에 50% 적용하여 지나치게 둥글어짐
