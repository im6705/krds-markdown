# 레이아웃 (Layout)

레이아웃은 정보 구조와 시각적 흐름을 체계적으로 구성하여 사용자가 콘텐츠를 쉽게 이해할 수 있도록 한다. 그리드, 브레이크포인트, 간격 설정을 통해 일관된 사용자 경험을 제공한다.

---

## 1. 그리드 시스템

### 그리드 구성 요소

| 요소 | 설명 |
|---|---|
| **Column (칼럼)** | 콘텐츠 영역을 수직으로 분할한 영역 |
| **Screen Margin (스크린 마진)** | 화면의 양쪽 가장자리의 사이의 공간 |
| **Gutter (가터)** | 칼럼 사이의 간격 |

### 스크린 마진

| 브레이크포인트 | 최소 스크린 마진 |
|---|---|
| large | **24px** |
| medium | **24px** |
| small | **16px** |

### 칼럼

| 브레이크포인트 | 칼럼 수 |
|---|---|
| large | **12~16개** |
| medium | **8~12개** |
| small | **4~6개** |

- 동일 화면에서 칼럼 수를 혼용하지 않는다
- 고정값 대신 **백분율**로 설정

### 가터

| 브레이크포인트 | 최소 | 최적 |
|---|---|---|
| large | 16px | **24px** |
| medium | 16px | **24px** |
| small | 16px | **16px** |

- 가터 크기는 **칼럼 너비의 절반 이하**로 설정

---

## 2. 브레이크포인트

브레이크포인트 단계는 **최소 3단계에서 6단계**로 설정.

### 전체 브레이크포인트 (6단계)

| 브레이크포인트 | 뷰포트 | 칼럼 수 (적정-최대) | 가터 너비 (최소-적정) | 최소 스크린 마진 |
|---|---|---|---|---|
| **xsmall** | ~359px | 4-6 | 16-16px | 16px |
| **small** | 360px~ | 4-6 | 16-16px | 16px |
| **medium** | 768px~ | 8-12 | 16-24px | 24px |
| **large** | 1024px~ | 12-16 | 16-24px | 24px |
| **xlarge** | 1280px~ | 12-16 | 16-24px | 24px |
| **xxlarge** | 1440px~ | 12-16 | 16-24px | 24px |

### 표준형 스타일 (4단계)

고정된 콘텐츠 영역을 사용하며, xsmall은 최적화에 포함되지 않는다.

| name | Viewport | 칼럼 수 | 가터 너비 | 최소 스크린 마진 |
|---|---|---|---|---|
| **small** | 360px~ | 4 | 16px | 16px |
| **medium** | 768px~ | 8 | 16px | 24px |
| **large** | 1024px~ | 12 | 24px | 24px |
| **xlarge** | 1280px~ | 12 | 24px | 24px |

### 확장형 스타일

- 3~6단계의 브레이크포인트 적용
- 뷰포트, 칼럼 수, 가터, 최소 스크린 마진 값은 준수

---

## 3. 콘텐츠 영역

### 제한된 콘텐츠 영역 (고정형)

- 표준형 스타일: **최대 1200px**
- 스크린 마진(24px) + 콘텐츠 너비(1200px) = **1248px** 기준
  - 디바이스 >= 1248px: 콘텐츠 영역 유지
  - 디바이스 < 1248px: 콘텐츠 영역 축소

### 풀 반응형 그리드 / 유연한 콘텐츠 영역 (가변형)

- 화면 크기에 따라 유동적으로 변화
- 큰 화면에서 콘텐츠 영역이 지나치게 넓어지지 않도록 가독성 고려

---

## 4. 서브 페이지 레이아웃

브레이크포인트 large 이상 (1024px 이상)에서 정보 구조에 따라 변형.

### 구성 요소

| # | 영역 | 설명 | 필수 여부 |
|---|---|---|---|
| 1 | **Header** | 최상단 영역 | 필수 |
| 2 | **Left menu** | 서브 화면 이동 메뉴 (사이드 메뉴) | 선택 |
| 3 | **Main contents** | 본문 콘텐츠 영역 | 필수 |
| 4 | **Right menu** | 콘텐츠 내 탐색 수단 | 선택 |
| 5 | **Footer** | 최하단 정보 영역 | 필수 |

### 메뉴 배치 유형

- **왼쪽 메뉴**: 사이드 메뉴 영역, 깊은 계층 구조 탐색에 적합. 최대 콘텐츠 너비 1200px (사이드 메뉴 포함)
- **오른쪽 메뉴**: 콘텐츠 내 탐색, 도움 패널, 빠른 메뉴. 콘텐츠 영역 밖에서 플로팅
- **양쪽 메뉴**: 큰 화면에서는 문제없으나, 작은 화면에서는 메뉴 배치 조정 필요

## 5. 간격 (Spacing)

**8-point grid system** 기준으로 모든 간격 설정.

### 간격 유형

| 유형 | 설명 |
|---|---|
| **Gap (갭)** | 컴포넌트 사이의 간격 (카드, 리스트, 그리드) |
| **Padding (패딩)** | 컴포넌트 내부의 여백 (내용과 경계선 사이) |
| **반응형 간격** | 화면 크기에 따라 조정되는 간격 |

### 반응형 간격 설정

| 모드 | 기준 사이즈 (콘텐츠 영역) | 범위 뷰포트 | 설명 |
|---|---|---|---|
| **Mobile** | 348px | small 360px~ / medium 768px~ | 태블릿 포함 소형 디바이스 |
| **PC** | 1200px | large 1024px~ / xlarge 1280px~ | 데스크탑 및 대형 화면 |

### 간격 적용 영역

| 영역 | 설명 |
|---|---|
| **Layout** | 기본 구조에서 사용되는 사이 간격 |
| **Contents** | 콘텐츠의 계층 구조를 위한 간격 |
| **Padding** | 컴포넌트 내부에 사용되는 패딩값 |

### 레이아웃 간격 (PC/Mobile)

| 요소 | PC | Mobile |
|---|---|---|
| Header - Breadcrumb | 24px (number-10) | 24px (number-10) |
| Left - Contents | 64px (number-18) | - |
| Contents - Right | 40px (number-14) | - |
| Contents - Footer | 64px (number-18) | 40px (number-14) |
| Breadcrumb - Contents(h1) | 40px (number-14) | 32px (number-11) |

### 컴포넌트 간격

| 컴포넌트 | 간격 토큰 | px 값 |
|---|---|---|
| **카드 리스트** (세로/가로/모듈형) | gap-7 | 24px |
| **인풋** (가로 조합) | gap-5 | 16px |
| **인풋** (세로 조합) | gap-5 ~ gap-7 | 16px ~ 24px |
| **체크박스/라디오** (세로 리스트) | gap-5 ~ gap-7 | 16px ~ 24px |
| **체크박스/라디오** (가로 리스트) | gap-7 이상 | 24px 이상 |

### 컴포넌트 내 패딩

| 컴포넌트 | 패딩 토큰 | 비고 |
|---|---|---|
| **카드, 모달, 인포박스, 알럿** | padding-card (padding8-10) | PC 넉넉, 모바일 적게 |
| **텍스트 입력 필드 내부 gap** | gap-3 | 8px |
| **텍스트 입력 필드 패딩** | padding-6 | 16px |

## 6. 사용 가이드

### 스크린 마진 최솟값 유지
- **모범 사례**: 디바이스별 스크린 마진 최소값을 유지
- **피해야 할 사례**: 모바일에서 디바이스 끝부분에 여백이 없어 터치가 어려움
