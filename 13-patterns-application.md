# 신청 서비스 패턴

신청은 사용자가 서비스 제공 기관에 요구 사항을 알리기 위해 서식을 작성하는 과업으로, 관련 법률에 명시된 종류의 민원 외에 사용자가 특정한 내용을 요청하는 모든 행동을 포함한다.

---

## 1. 신청 유형

| 유형 | 설명 | 예시 |
|---|---|---|
| **제출** | 정해진 서식에 따라 신청서를 작성하여 제출 | 복지 급여 신청, 전입신고, 귀국신고 |
| **조회** | 자격, 문서 효력, 처분 결과 등을 확인 | 문서 진위 확인, 세액 조회 |
| **발급** | 증명 서류, 자격 번호 등을 발급 | 주민등록 등·초본 발급, 개인통관번호 발급 |
| **예약** | 일정 기간 동안의 재화/서비스에 대한 이용 신청 | 도서 예약, 좌석 예약, 관람 예약/예매 |

---

## 2. 이용 상황별 플로(Flow)

| 단계 | 과정 | 세부 |
|---|---|---|
| 1 | **신청 대상 탐색** | 서비스 목록 탐색, 대상 선택하기, 신청하기 |
| 2 | **신청서 작성** | 단계의 진행과 탐색, 신청 정보 입력, 자동 입력, 작성 도움말 확인, 매뉴얼 확인, 서류 제출, 여러 건 신청하기, 임시 저장 |
| 3 | **확인/확정** | 신청 내용 최종 확인, 신청서 제출 확정, 신청서 작성 취소 |
| 4 | **신청 결과 확인** | 결과 화면 찾기, 결과 목록 탐색, 진행 상태 확인 |

---

## 3. 단계별 상세

### 3.1 신청 대상 탐색

#### 구조

| # | 요소 | 설명 |
|---|---|---|
| 1 | **필터링·정렬 컨트롤** | 신청 서비스 목록을 필터링·정렬하는 데 사용되는 컨트롤 |
| 2 | **페이지네이션** | 신청 서비스 목록을 탐색하는 데 사용되는 컨트롤 |
| 3 | **항목** | 정보를 식별하기 위한 콘텐츠 집합으로 개별 항목에 대해 실행할 기능 관련 버튼, 상세 정보를 확인할 수 있는 탐색 링크가 포함될 수 있음 |
| 3-a | 제목 | 서비스명을 보여주는 텍스트. 상세 화면으로 이동하기 위한 링크로 사용됨 |
| 3-b | 꺾쇠/화살표 | 제목이 링크로 작동함을 안내하는 시각적 단서 |
| 3-c | 미리보기/요약 | 서비스에 대한 기본적인 정보를 요약하여 보여주는 텍스트 |
| 3-d | 배지 | 서비스의 이용 가능 상태에 대한 메타 데이터 (예정됨/접수 중/마감됨) |
| 3-e | 메타 데이터 | 신청 서비스에 부여된 여러 데이터 속성 (분야, 연령, 소득 등) |
| 3-f | 저장 컨트롤 | 관심 있는 신청 정보를 모아보기 위해 개인화된 기록을 저장하는 컨트롤 |
| 3-g | 액션 버튼 | 신청하기 버튼 등 항목에 대해 수행할 수 있는 기능 사용을 유도하기 위한 컨트롤 |

#### 사용성 가이드라인

| # | 가이드라인 | 적용 수준 |
|---|---|---|
| 01 | 신청 서비스의 특성에 따른 적절한 서비스 목록 정렬 방식을 사용한다. 상시 신청 가능: 가나다순. 한시적/인원 제한: 현재 신청 가능한 서비스를 우선 배치 | 권장 |
| 02 | 목록을 적절한 방식으로 군집화하여 메타 데이터를 부여한다 | 권장 |
| 03 | 신청 상태 정보를 명확하게 인지할 수 있도록 표현한다. 한시적/인원 제한 서비스인 경우 배지를 반드시 제공 | 필수 |
| 04 | 미리보기/요약에 신청 가능 기한 정보를 제공한다 | 권장 |
| 05 | 제목에 공식적인 서비스 명칭을 사용한다. 임의의 축약어를 사용하거나 공식 명칭과 다른 단어 사용 시 혼동을 줄 수 있다 | 필수 |
| 06 | 제목에 말줄임표를 사용하지 않는다. 정확한 서비스 명칭을 확인할 수 있도록 제목 텍스트를 자르지 않는다 | 필수 |
| 07 | 미리보기/요약 텍스트는 간결하게 작성한다 | 권장 |
| 08 | 상세 화면을 거치지 않고 신청 과정에 바로 접근할 수 있는 액션 버튼을 제공한다 | 권장 |
| 09 | 외부 서비스로 이동하거나 새 창을 실행하는 액션 버튼에 명확한 시각적 단서를 제공한다 | 권장 |

#### 접근성 가이드라인

- 공유/저장 컨트롤과 액션 버튼에 명확한 접근 가능한 이름을 제공한다. 스크린 리더 사용자가 컨트롤 요소를 단위로 탐색 시, 동일한 레이블을 가진 컨트롤이 다수 제공되면 대상 정보를 파악하기 어렵다. title 속성 또는 aria-describedby 속성을 활용하여 접근 가능한 이름이 변별될 수 있도록 한다 (KWCAG 2.2 적절한 링크 텍스트, WCAG 2.1 Headings and Labels (AA), WCAG 2.1 Label in Name (A), WCAG 2.1 Name, Role, Value (A))

#### 플랫폼에 대한 고려 사항

- 검색어 입력창에 음성입력 버튼을 제공한다
- 모바일 앱에서 필터 컨트롤은 모달이나 바텀시트를 활용하여 표현한다
- 모바일 앱에서는 화면 크기를 고려하여 상세 검색 컨트롤을 숨김 처리할 수 있다

### 3.2 서비스 정보 확인

#### 구조

| # | 요소 | 설명 |
|---|---|---|
| 1 | **제목** | 본문 제목으로 신청 서비스명이 텍스트로 제공됨 |
| 2 | **요약 정보** | 신청 서비스의 특징이나 목적을 요약하여 설명하는 2~3줄의 텍스트 |
| 3 | **개요 정보** | 신청 서비스의 핵심 정보에 대한 개요 목록 |
| 4 | **상세 정보** | 신청 서비스 상세 정보를 전달하기 위한 콘텐츠 섹션 |
| 5 | **부가 정보** | 서비스나 신청 과정 진입 전에 반드시 확인해야 할 정보는 아니지만 서비스에 대한 이해를 돕기 위해 제공되는 콘텐츠 섹션 |
| 6 | **콘텐츠 변경 이력** | 정보의 업로드일, 변경일, 변경된 내용에 대한 로그 기록 |
| 7 | **콘텐츠 내 탐색** | 본문의 콘텐츠 섹션의 구조를 파악하고 원하는 섹션으로 이동할 수 있도록 도와주는 탐색 수단 |
| 8 | **액션 버튼** | 신청 단계로 이동하거나 신청할 수 있는 외부 서비스 채널로 이동할 수 있는 링크 |

#### 사용성 가이드라인

| # | 가이드라인 | 적용 수준 |
|---|---|---|
| 01 | 본문 콘텐츠의 제목을 신청 서비스명으로 제공한다 | 권장 |
| 02 | 상세 정보 콘텐츠는 간결하며 이해하기 쉽게 작성한다 | 필수 |
| 03 | 상세 정보는 정확하고 최신화된 상태로 제공한다 | 필수 |
| 04 | 서비스를 신청할 수 있는 모든 채널과 제약 사항에 대한 정보를 제공한다 | 필수 |
| 05 | 온라인으로 신청할 수 없는 상황에 대해 명확하게 인지 가능하도록 표현한다 | 필수 |
| 06 | 사용자가 신청 과업을 완료하는 데 필요한 모든 서식과 서류에 대해 안내하고 빠르게 접근할 수 있는 수단을 제공한다 | 필수 |
| 07 | 신청 과정과 처리 절차에 대한 정보를 제공한다 | 필수 |
| 08 | 일반적으로 신청 과정에 소요되는 기간 정보를 절차별로 안내한다 | 권장 |
| 09 | 사용자에게 도움을 줄 수 있는 다양한 부가 정보를 제공한다 | 권장 |
| 10 | 기본 정보는 한 화면에 확인 가능하도록 제공한다 | 필수 |
| 11 | 기본 정보는 첫 번째 탭에서 확인 가능하도록 제공한다 | 필수 |
| 12 | 부가 정보는 사용자가 필요에 따라 상세 내용을 확인할 수 있게 제공하는 방안을 고려한다 | 권장 |
| 13 | 모든 링크는 실행하였을 때, 링크 레이블에 명시된 적절한 화면으로 이동해야 한다 | 권장 |
| 14 | 목록 이동 버튼을 누르거나 사용자 에이전트에서 뒤로가기 동작을 실행하였을 때 사용자의 탐색 맥락이 유지되어야 한다 | 권장 |

### 3.3 유의 사항/자격 확인

#### 구조

| # | 요소 | 설명 |
|---|---|---|
| 1 | **배지** | 신청 서비스의 주요 메타 데이터를 요약하여 제공 |
| 2 | **안내 정보** | 신청서 작성 전 유의해야 할 사항, 제출해야 할 서류 등의 정보를 안내 |
| 2-a | 취소/수정 가능 여부 | |
| 2-b | 전자서명/본인인증 절차의 포함 여부 | |
| 2-c | 제출할 증빙서류 | |
| 2-d | 기타 안내사항 | |
| 3 | **디스클로저** | 부가적으로 참고 가능한 정보를 제공하는 확장 가능한 영역 |
| 4 | **액션 버튼** | 다음 단계로 이동하기 위한 액션 버튼 |

#### 유형

| 유형 | 설명 |
|---|---|
| **안내 섹션** | 신청서 입력폼 상단에 직접 제공되는 형태. 안내 정보의 중요도에 따라 디스클로저 컴포넌트와 함께 사용할 수 있다 |
| **단일 화면** | 신청의 첫 단계로 '유의 사항 및 자격 확인'을 진행. 사용자가 서비스를 신청하기에 적합한 조건을 확인하기 위한 입력폼 또는 유의 사항에 대한 안내를 제공 |

#### 사용성 가이드라인

| # | 가이드라인 | 적용 수준 |
|---|---|---|
| 01 | 유의 사항 및 자격 확인은 신청서 작성 과업의 가장 첫 단계로 제공한다 | 권장 |
| 02 | 신청서 작성 중간이나 최종 단계에서 작성을 중단해야 하는 상황에 대해 사용자에게 명확하게 안내한다 | 필수 |
| 03 | 중요도와 계층 구조에 따라 정보를 직관적으로 인지할 수 있도록 표현한다 | 권장 |
| 04 | 신청서 작성의 단계 중에 필요한 정보, 신청 완료 이후 필요한 정보는 포함하지 않는다 | 권장 |

### 3.4 신청서 작성

#### 구조

| # | 요소 | 설명 |
|---|---|---|
| 1 | **제목** | 입력폼의 목적에 대한 설명 |
| 2 | **설명** (선택) | 신청의 목적에 대한 부가적인 설명이나 사용자가 데이터를 입력할 때 유의해야 하는 사항 등 총체적인 안내를 제공하는 문장 |
| 3 | **입력폼** | 신청서 작성에 필요한 데이터를 입력하기 위한 양식 |
| 4 | **단계 표시기** | 작성의 단계가 있는 신청에서 현재 진행 중인 단계를 안내하기 위해 제공 |
| 5 | **도움 패널** | 신청서 작성에 필요한 도움말, 참고 정보 링크, 문의처 정보가 제공 |
| 6 | **어노테이션 아이콘** | 도움 패널을 실행하는 데 사용되는 버튼 |

#### 사용성 가이드라인

| # | 가이드라인 | 적용 수준 |
|---|---|---|
| 01 | 한 단계 내에서의 신청 서식 섹션을 탭이나 아코디언으로 구분하지 않는다 | 필수 |
| 02 | 값의 변경이 이전에 작성한 항목 또는 이전 단계에 영향을 미치는 경우, 이전 입력 항목 또는 이전 단계에 포함시킨다 | 권장 |
| 03 | 값의 변경이 이전에 작성한 항목 또는 이전 단계에 영향을 미치는 상황에 대해 사용자에게 안내해야 한다 | 필수 |
| 04 | 행정정보 공동이용 시스템을 활용하여 사용자의 구비서류 첨부를 최소화한다 | 권장 |
| 05 | 여러 단계로 구성되어 있거나 작성에 시간이 걸리는 신청서 작성 과정에 임시 저장 기능을 제공한다 | 권장 |
| 06 | 입력 항목에 대한 레이블에 설명을 제공한다 | 권장 |
| 07 | 신청서 작성에 대해 도움받을 수 있는 다양한 수단을 제공한다 | 필수 |
| 08 | 도움말은 이용 맥락에 맞게 제공한다 | 권장 |
| 09 | 도움말은 정확한 내용으로 제공한다 | 필수 |
| 10 | 초기화 버튼은 다른 버튼과 명확하게 구분한다 | 필수 |

#### 접근성 가이드라인

- 현재 단계를 스크린 리더로 확인할 수 있도록 한다 (KWCAG 2.2, WCAG 2.1 Name, Role, Value)
- 신청서 섹션에 제목을 제공하고 프로그램적으로 탐지할 수 있도록 한다 (KWCAG 2.2 제목 제공, WCAG 2.1 Info and Relationships, WCAG 2.1 Labels or Instructions)

#### 구조

| # | 요소 | 설명 |
|---|---|---|
| 1 | **제목** | 요청하는 행동에 대한 간단한 요약 정보를 제공 |

| 4 | **액션 컨트롤** | 신청서의 제출을 확정하거나 중단하고 신청서 작성 단계로 돌아가는 액션을 위한 버튼 그룹 |

#### 유형
