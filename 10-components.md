# 컴포넌트 개요

사용자 인터페이스의 가장 작은 단위로 과업에 상관없이 일관성 있게 사용되는 공통 요소에 대한 가이드이다. 각 컴포넌트의 특성과 상태, 역할에 따른 유형을 제시하며, 사용성/접근성/상호작용 가이드라인 및 플랫폼 고려 사항을 기술한다.

---

## 1. 컴포넌트 분류 (10개 범주)

| # | 범주 | 컴포넌트 |
|---|---|---|
| 1 | **아이덴티티** | 공식 배너(Masthead), 운영기관 식별자(Identifier), 헤더(Header), 푸터(Footer) |
| 2 | **탐색** | 건너뛰기 링크, 메인 메뉴, 브레드크럼, 사이드 메뉴, 콘텐츠 내 탐색, 페이지네이션 |
| 3 | **레이아웃/표현** | 구조화 목록, 긴급 공지, 달력, 디스클로저, 모달, 배지, 아코디언, 이미지, 캐러셀, 탭, 표 |
| 4 | **액션** | 링크, 버튼 |
| 5 | **선택** | 라디오 버튼, 셀렉트, 체크박스, 태그 |
| 6 | **피드백** | 단계 표시기, 스피너 |
| 7 | **도움** | 도움 패널, 따라하기 패널, 맥락적 도움말, 코치마크 |
| 8 | **입력** | 날짜 입력 필드, 텍스트 영역, 텍스트 입력 필드, 파일 업로드 |
| 9 | **설정** | 언어 변경, 화면 크기 조정 |
| 10 | **콘텐츠** | 접근 가능한 미디어, 숨긴 콘텐츠 |

## 2. 아이덴티티 컴포넌트

### 2.1 공식 배너 (Masthead)

대한민국 정부 조직 및 관련 기관에서 운영/관리하는 공식 디지털 정부서비스임을 식별할 수 있도록 제공하는 정보 배너.

**구조**: 컨테이너 + 텍스트 ("이 누리집은 대한민국 공식 전자정부 누리집입니다.")

#### 사용성 가이드라인
1. 배너는 **모든 화면의 최상단**에 제공한다
2. 배너가 지나치게 주의를 끌지 않도록 표현한다
3. 배너 텍스트와 스타일을 **변형하지 않는다**
4. 공식 디지털 정부서비스가 아닌 사이트에서는 배너를 사용하지 않는다

#### 접근성 가이드라인
- 건너뛰기 링크는 공식 배너 **이전에** 제공 (WCAG 2.1 Bypass Blocks (A))

---

### 2.2 운영기관 식별자 (Identifier)

서비스 운영기관을 안내하는 요소. 공식 배너, 푸터와 함께 서비스의 일관성/브랜드를 확인하는 핵심 요소.

**구조**: 컨테이너 + 운영기관 로고 + 안내 텍스트

**안내 텍스트 예시**: "(운영기관 로고) 이 누리집은 [운영기관명]에서 운영하는 누리집입니다."

#### 사용성 가이드라인
1. 식별자가 지나치게 주의를 끌지 않도록 표현
2. **서비스 로고가 아닌 운영 기관의 로고**를 제공
3. 안내 텍스트와 요소 배치를 변경하지 않는다
4. 공식 서비스가 아닌 사이트에서는 사용하지 않는다

#### 접근성 가이드라인
- 식별자 영역은 구조적으로 **푸터 내부에 포함** (WCAG 2.1 Info and Relationships (A))
- 로고 이미지에 대체 텍스트 제공 (WCAG 2.1 Non-text Content (A))

---

### 2.3 헤더 (Header)

웹사이트 최상단 영역. 브랜드 이미지를 전달하는 핵심 영역. 통합검색, 메인 메뉴 등 핵심 탐색 수단 제공.

**유형**: 세로형, 가로형

**구조** (7개 요소):
1. **건너뛰기 링크**: 키보드 사용자를 위한 보조 탐색 수단
2. **공식 배너**: 정부 서비스 식별 배너
3. **서비스 아이덴티티**: 기관/서비스 로고 및 슬로건
4. **유틸리티 링크 그룹**: 부차적이지만 빈번하게 사용되는 기능
5. **드롭다운 메뉴**: 유틸리티 링크가 5개 이상일 때
6. **아이콘과 레이블**: 통합검색, 로그인, 회원가입, 개인 메뉴
7. **메인 메뉴**: 사이트 일차 탐색 인터페이스

#### 사용성 가이드라인 (11개)
1. 헤더의 스타일 수정을 최소화한다
2. 정부 로고는 항상 헤더의 **왼쪽 상단**에 제공
3. 내부 컴포넌트를 일관된 순서로 배치: 건너뛰기 링크 > 서비스 아이덴티티 > 유틸리티 > 검색 > 메인 메뉴
4. 핵심적인 정보만 간결하게 제공
5. 유틸리티 링크 그룹은 헤더 **우측 상단**에 제공
6. 유틸리티 링크 그룹을 **디바이더**로 구분
7. 유틸리티 링크가 **5개 이상** 필요한 경우 드롭다운 메뉴 사용
8. 언어 설정 드롭다운에 **국기를 사용하지 않는다** (언어 이름을 해당 언어로 제공)
9. 아이콘 버튼/링크에 텍스트 레이블 제공 (검색, 로그인처럼 명확한 경우 생략 가능)
10. 메인 메뉴와 검색 위치는 웹사이트 목적에 적합하게 제공
11. 스크롤 시 헤더를 뷰포트 상단에 **고정 가능** (2행 초과 시: 스크롤 내릴 때 비고정, 올릴 때 고정. 고정 시 공식 배너는 제외)

#### 플랫폼 고려 사항
- 화면 너비 부족 시 **햄버거 메뉴** 형태로 제공 가능

#### 접근성 가이드라인
- 건너뛰기 링크를 최상단 배치 (WCAG 2.1 Bypass Blocks (A))
- 로고 이미지에 대체 텍스트 제공, '로고'라는 단어 포함 금지 (WCAG 2.1 Non-text Content (A))

---

### 2.4 푸터 (Footer)

화면의 가장 마지막 요소. 추가적인 탐색 수단과 유용한 링크 제공.

**구조** (6개 요소):
1. 컨테이너
2. 서비스 로고
3. 연락처 (주소, 이메일, 대표 전화번호, 민원 전화번호)
4. 유틸리티 링크 (오시는 길, 관련 기관, 소셜 채널)
5. 정책 링크 (웹 접근성, 개인정보처리방침)
6. 저작권 정보

#### 사용성 가이드라인 (7개)
1. 모든 화면에 푸터를 **일관된 위치**에 제공 (본문 하단. 본문이 짧은 경우 뷰포트 하단 고정)
2. 푸터 섹션 정보는 **일관된 순서**: 서비스 로고 > 연락처 > 유틸리티 링크 > 정책 링크 > 저작권 정보
3. **개인정보처리방침을 반드시 표시** (글자 크기, 색상, 굵기로 다른 고지사항과 구분)
4. 전자상거래 포함 시 **이용약관을 반드시 표시**
5. 사용자가 빈번하게 찾는 링크 배치 (사이트맵 등, 목적별/유형별 군집화)
6. 서비스 내비게이션은 기본 내비게이션과 **동일한 구조 및 레이블**
7. 연락처/링크는 항상 **최신 상태**로 유지
